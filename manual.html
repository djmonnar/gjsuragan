시스템 접속 방법
🌐 접속 방법

GitHub Pages 주소를 브라우저에 입력하면 로그인 화면이 나타납니다.
북마크(즐겨찾기)에 저장해두면 편리합니다.

  1. 비밀번호 입력
    가운데 입력창에 비밀번호를 입력합니다. 비밀번호는 대소문자, 특수문자를 구별합니다.
  2. 로그인 버튼 클릭 또는 Enter
    버튼을 클릭하거나 키보드 Enter를 누르면 로그인됩니다.
  3. 8시간 자동 유지
    로그인 후 8시간 동안은 페이지를 닫고 다시 열어도 자동으로 로그인 상태가 유지됩니다.
⚠️
비밀번호를 5회 연속 틀리면 잠금됩니다. 페이지를 새로고침(F5)하면 다시 시도할 수 있습니다.
ℹ️
로그아웃은 화면 오른쪽 상단의 '로그아웃' 버튼을 누르면 됩니다.

대시보드
전체 현황을 한눈에 파악하는 메인 화면
📅 날짜 선택 네비게이터

대시보드 상단의 ◁ ▷ 화살표로 날짜를 하루씩 이동하거나, 날짜를 직접 클릭해서 선택할 수 있습니다.

오늘 버튼을 누르면 바로 오늘 날짜로 돌아옵니다.

👉 특정 날짜의 배송 현황을 미리 확인할 때 유용합니다.

카드설명
전체 고객등록된 총 고객 수
활성 구독현재 정기배송 중인 고객 수
오늘 배송선택한 날짜에 출고해야 할 건 수
이번 주오늘부터 7일간 예정된 배송 건 수
갱신 임박잔여 배송 3회 이하인 정기 고객 수
⚠️ 갱신 임박 경고

잔여 배송이 3회 이하인 고객이 있으면 노란색 경고 배너가 나타납니다. '확인하기' 버튼을 눌러 해당 고객에게 갱신 안내를 해주세요.

✅
대시보드는 실시간으로 자동 업데이트됩니다. 다른 직원이 배송 처리를 하면 내 화면에도 즉시 반영됩니다.

배송 관리
날짜별 배송 목록 조회 및 처리
📋 배송 목록 보는 법

상단 날짜 네비게이터로 날짜를 선택하면 그날 출고해야 할 고객 목록이 자동으로 표시됩니다.

정기배송은 조리일(출고일) 기준으로 표시됩니다. 예: 수·금 조리 → 목·토 도착 고객은 수요일과 금요일에 목록에 나타납니다.

선택주문은 배송 예정일로 지정된 날에 목록에 나타납니다.

컬럼내용
고객명주문자 이름
연락처수령자 전화번호
배송지배송 주소 (마우스를 올리면 전체 주소 표시)
세트/유형주문 상품과 정기/선택 구분
배송일정예: 수·금 조리 → 목·토 도착
현관번호현관 비밀번호 또는 안내
요청사항고객 배송 메모
수량선택주문 시 수량 표시
잔여정기배송 남은 횟수 게이지
✓ 배송완료 처리하기
  1. 개별 처리
    각 행 오른쪽의 ✓ 버튼을 누르면 해당 고객만 배송완료 처리됩니다. 정기배송은 잔여 횟수가 1 감소합니다.
  2. 전체 처리
    상단의 ✓ 전체 완료 버튼을 누르면 목록에 있는 모든 고객을 한번에 처리합니다. 확인 창이 뜨니 실수 걱정은 없어요.
  3. 완료 취소 (실수했을 때)
    고객 관리 탭 → 해당 고객 클릭 → 상세 패널 배송 이력에서 ↩ 버튼을 누르면 취소됩니다.
ℹ️
직배송 고객은 직배송 뱃지가 표시됩니다. 사천·진주 지역 등 당일 직접 배달하는 고객입니다.

고객 관리
고객 검색, 조회, 수정, 삭제
🔍 고객 검색하기

상단 검색창에 이름, 연락처, 주소, 주문번호 중 아무거나 입력하면 즉시 필터링됩니다.

드롭다운 필터로 세트, 유형(정기/선택), 상태(구독중/정지/종료), 배송방식(직배송/택배)을 조합해서 검색할 수 있습니다.

👤 고객 상세 정보

고객 행을 클릭하면 오른쪽에 상세 패널이 열립니다. 여기서 모든 정보를 확인하고 다양한 작업을 할 수 있습니다.

버튼기능
✏ 수정고객 정보, 세트, 상태 등 수정
✓ 배송완료오늘 날짜로 배송완료 처리
⏸ 일시정지정기배송 일시중단 (잔여 횟수 유지)
▶ 재개일시정지된 구독 다시 활성화
＋ 횟수 충전정기 갱신 시 잔여 횟수 추가
📅 배송일 변경선택주문 배송 예정일 변경
↩ 이력 취소배송완료 실수 시 되돌리기
🗑 삭제고객 완전 삭제 (복구 불가)
💰 정기배송 갱신 방법
  1. 갱신 임박 고객 확인
    대시보드 또는 고객 목록에서 잔여 3회 이하 고객을 확인합니다.
  2. 고객 클릭 → 상세 패널
    해당 고객을 클릭해서 상세 패널을 엽니다.
  3. ＋ 횟수 충전 클릭
    충전할 횟수를 입력합니다 (보통 12회). 현재 잔여 횟수에 더해집니다.
⬇ CSV 전체 백업

고객 관리 탭 오른쪽 상단의 ⬇ CSV 백업 버튼을 누르면 전체 고객 정보가 엑셀에서 열 수 있는 CSV 파일로 저장됩니다.

파일명: 궁중수라간_고객백업_날짜.csv — 정기적으로 백업해두는 것을 권장합니다.

⚠️
삭제는 복구가 불가합니다. 구독이 끝난 고객은 삭제 대신 상태를 '종료'로 변경해두는 것이 좋습니다. 이력이 보존됩니다.
✏️
고객 직접 등록
신규 고객을 수동으로 등록하는 방법
📦 정기배송 등록
  1. ＋ 신규 등록 버튼 클릭
    헤더 오른쪽 상단 또는 고객 관리 탭의 버튼을 누릅니다.
  2. 📦 정기배송 탭 선택 확인
    모달 상단에서 정기배송 탭이 선택되어 있는지 확인합니다.
  3. 기본 정보 입력
    이름(필수), 연락처(필수), 주소(필수), 현관번호, 요청사항을 입력합니다.
  4. 구독 설정
    세트 선택(A/B/C) → 배송 주기(주1/2/3회) → 배송 일정 선택 → 총 구독 횟수 → 시작일을 설정합니다.
  5. 등록 완료 클릭
    모든 항목 입력 후 하단 '등록 완료' 버튼을 누릅니다.
🛍 선택주문 등록

모달에서 🛍 선택주문 (1회) 탭을 선택합니다. 정기배송과 달리 아래 항목만 설정합니다:

  • 상품 — 반찬 세트(A/B/C) 또는 단품(돼지양념갈비, LA갈비)
  • 배송 예정일 — 실제 배송 나가는 날짜
  • 수량 — ＋/− 버튼으로 조절
🚗 직배송 설정

사천·진주 지역 등 직접 배달하는 고객은 "직배송 (당일 배송)" 체크박스에 체크합니다.

직배송 체크 시 배송 목록에 직배송 뱃지가 표시되고, 택배 내보내기 목록에서도 별도 섹션으로 분리됩니다.

배송 일정조리일도착일
주 1회 — 월 조리월요일화요일
주 1회 — 수 조리수요일목요일
주 1회 — 금 조리금요일토요일
주 2회 — 수·금 조리수, 금목, 토
주 2회 — 월·수 조리월, 수화, 목
주 3회 — 월·수·금 조리월, 수, 금화, 목, 토
ℹ️
배송 목록은 조리일(출고일) 기준으로 표시됩니다. 수·금 조리 고객은 수요일과 금요일에 배송 목록에 나타납니다.

일괄 등록
텍스트 붙여넣기 또는 엑셀로 한번에 등록
📋 방법 1 — 주문 텍스트 붙여넣기

스마트스토어 주문 내역을 그대로 복사해서 붙여넣기하면 이름, 연락처, 주소, 상품, 수량 등을 자동으로 인식합니다.

  1. 스마트스토어에서 주문 내역 복사
    주문 상세 화면에서 전체 내용을 드래그해서 복사(Ctrl+C)합니다.
  2. 텍스트 입력창에 붙여넣기
    📥 일괄 등록 탭 왼쪽 입력창에 붙여넣기(Ctrl+V)합니다.
  3. 🔍 자동 분석 클릭
    버튼을 누르면 이름, 연락처, 주소, 상품, 수량이 자동으로 인식되어 결과가 표시됩니다.
  4. 등록하기 클릭 → 확인 모달에서 수정 후 등록
    인식된 정보가 모달에 채워집니다. 잘못된 항목이 있으면 직접 수정하고 '등록 완료'를 누릅니다.
🔄 복수 주문 (날짜 다른 2건 이상)

한 명의 고객이 날짜가 다른 2개 이상의 주문을 한번에 한 경우, 자동으로 감지해서 각 주문을 별도로 표시합니다.

예시: 202603258223322-001 3월 27일 금요일 → B세트, 1개 202603258223322-002 3월 30일 월요일 → C세트, 1개 → 주문 1, 주문 2로 각각 표시됨 → 등록하기 누르면 2건이 별도로 Firebase에 저장됨

모달에서 각 주문의 상품·날짜·수량을 수정할 수 있고, ✕ 버튼으로 특정 주문을 제외할 수도 있습니다.

📊 방법 2 — 엑셀 일괄 업로드

스마트스토어에서 다운받은 주문 엑셀 파일을 그대로 업로드하면 수령자명, 연락처, 주소, 배송메모를 자동으로 읽어옵니다.

  1. 엑셀 파일 업로드
    드롭존에 파일을 드래그하거나 클릭해서 선택합니다. (.xlsx, .xls 지원)
  2. 세트 확인 (필수!)
    미리보기에서 각 고객의 세트를 확인하고 선택합니다. 세트가 선택되지 않으면 등록이 안됩니다. 상품명에 A/B/C세트가 포함되면 자동 인식됩니다.
  3. ✓ 전체 등록 클릭
    확인 후 '전체 등록' 버튼을 누르면 모든 고객이 한번에 등록됩니다.
⚠️
✕ 버튼으로 특정 행을 제외하고 등록할 수 있습니다. 취소나 환불 건은 미리 제외하세요.

택배사 전달
배송 목록 생성 및 전달 파일 만들기
📋 배송 목록 생성하기
  1. 배송일 선택
    택배사에 전달할 배송 날짜를 선택합니다.
  2. 세트/유형 필터 (선택사항)
    특정 세트나 정기/선택 유형만 필터링할 수 있습니다. 전체 보내려면 '전체'로 유지합니다.
  3. 📋 목록 생성 클릭
    버튼을 누르면 오른쪽에 미리보기, 왼쪽에 텍스트 목록이 생성됩니다.
  4. 📋 복사 또는 ⬇ 파일저장
    복사 버튼으로 클립보드에 복사해서 카카오톡 등으로 전달하거나, 파일저장으로 .txt 파일을 저장합니다.

🔄 실시간 동기화

직원 3명이 동시에 접속해도 괜찮습니다. Firebase가 모든 변경사항을 실시간으로 동기화합니다.

📱 모바일 사용

스마트폰에서도 동일하게 사용할 수 있습니다. 주소를 홈화면에 추가해두면 앱처럼 바로 접속됩니다.

🔍 주문번호 검색

고객 관리 탭 검색창에 스마트스토어 주문번호를 입력하면 바로 해당 고객을 찾을 수 있습니다.

⏸ 일시정지 활용

고객이 여행 등으로 잠시 쉬는 경우, 삭제하지 말고 일시정지를 사용하세요. 잔여 횟수가 그대로 보존됩니다.

💾 정기 백업

월 1회 정도 CSV 백업을 해두는 것을 권장합니다. 고객 관리 탭 → ⬇ CSV 백업 버튼.

🚨 문제 상황 대처법
상황해결 방법
로그인이 안 될 때 비밀번호 5회 실패 시 F5(새로고침) 후 재시도. 영어 대소문자·특수문자 확인.
로그인 없이 들어가질 때 F12 → Console에 localStorage.clear(); location.reload(); 입력 후 Enter.
🔴 연결 오류 표시 인터넷 연결 확인 후 페이지 새로고침(F5). Firebase 서버 점검 중일 수 있습니다.
배송 목록에 고객이 안 보일 때 고객 상태가 '활성'인지, 잔여 횟수가 남아있는지, 배송일정이 올바른지 확인.
실수로 배송완료 처리했을 때 고객 상세 패널 → 배송 이력 → ↩ 버튼으로 취소.
텍스트 파싱이 안 될 때 인식 실패 항목은 모달에서 직접 수정 가능. 또는 ＋ 신규 등록으로 수동 입력.
