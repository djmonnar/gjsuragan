# 궁중수라간 카카오 챗봇 배송조회 세팅 가이드

## 목표

카카오톡 챗봇에서 아래처럼 입력하면 Firebase `customers` 컬렉션 기준으로 배송 정보를 바로 확인합니다.

- `오늘배송`
- `내일배송`
- `요약`
- `고객검색 홍길동`
- `고객검색 0101234`
- `6월 12일 배송`

## 추가된 파일

- `appscript/KakaoWebhook.gs`

이 파일은 기존 `appscript/Code.gs`와 같은 Google Apps Script 프로젝트에 추가하는 용도입니다.
기존 `Code.gs`의 `CONFIG.PROJECT_ID`, `getFirebaseToken()` 함수를 재사용합니다.

## 중요 보안 메모

고객명, 전화번호, 주소가 카카오톡으로 노출되므로 반드시 관리자 PIN 인증을 사용해야 합니다.

Apps Script에서 아래 값을 스크립트 속성에 등록하세요.

- 속성명: `KAKAO_ADMIN_PIN`
- 값: 원하는 관리자 PIN 숫자 또는 문자열

예: `8274`

챗봇에서는 최초 1회 아래처럼 입력합니다.

```text
인증 8274
```

인증 후 6시간 동안 같은 카카오 사용자 기준으로 조회가 허용됩니다.

## Google Apps Script 적용 순서

1. 기존 궁중수라간 아임웹/Firebase 연동 Apps Script 프로젝트를 엽니다.
2. 새 스크립트 파일을 만들고 이름을 `KakaoWebhook.gs`로 지정합니다.
3. GitHub의 `appscript/KakaoWebhook.gs` 전체 내용을 복사해 붙여넣습니다.
4. 프로젝트 설정에서 스크립트 속성 `KAKAO_ADMIN_PIN`을 등록합니다.
5. 배포 > 새 배포 > 웹 앱을 선택합니다.
6. 실행 사용자: `나`
7. 액세스 권한: 카카오 챗봇 스킬 서버가 호출할 수 있는 값으로 설정합니다.
8. 배포 후 웹 앱 URL을 복사합니다.
9. 웹 앱 URL을 브라우저에서 열었을 때 `GJSURAGAN Kakao webhook OK`가 보이면 기본 연결은 성공입니다.

## 카카오 챗봇 관리자센터 세팅

### 1. 스킬 만들기

- 스킬명: `궁중수라간 배송조회`
- URL: Apps Script 웹 앱 URL
- Method: `POST`
- 응답 타입: 스킬 응답 JSON 사용

### 2. 블록 예시

아래 블록을 만들고 모두 같은 스킬을 연결해도 됩니다.

| 블록명 | 사용자 발화 예시 | 파라미터 |
|---|---|---|
| 오늘배송 | 오늘배송, 오늘 배송, 금일배송 | 없음 |
| 내일배송 | 내일배송, 내일 배송 | 없음 |
| 배송요약 | 요약, 현황, 배송요약 | 없음 |
| 고객검색 | 고객검색 @keyword, 고객 @keyword | `keyword` |
| 날짜배송 | @date 배송, 6월 12일 배송 | `date` |

처음에는 파라미터 없이 발화 텍스트만으로도 동작하게 만들어두었습니다.

## 현재 구현된 명령

| 명령 | 동작 |
|---|---|
| 인증 PIN | 관리자 인증 처리 |
| 오늘배송 | 오늘 날짜 배송 목록 조회 |
| 내일배송 | 내일 날짜 배송 목록 조회 |
| 모레배송 | 모레 날짜 배송 목록 조회 |
| 요약 | 오늘/내일/활성 정기/활성 선택주문/확인 필요 건수 조회 |
| 고객검색 키워드 | 이름, 전화번호, 주소, 주문번호, 메모, 요청사항, 일정명에서 검색 |
| YYYY-MM-DD 배송 | 해당 날짜 배송 조회 |
| M월 D일 배송 | 올해 기준 해당 날짜 배송 조회 |

## 주의사항

- 이 기능은 고객정보를 카카오톡으로 내보내는 기능이므로, 일반 고객용 챗봇이 아니라 내부 관리자용 챗봇에서만 사용해야 합니다.
- Apps Script 웹 앱 URL이 외부에 노출되면 PIN을 모르는 사람도 인증 시도를 할 수 있습니다.
- GitHub 공개 저장소에는 Firebase 서비스 계정 private key, 아임웹 secret key를 올리지 않는 것이 안전합니다.
- 기존 `Code.gs`에 키가 하드코딩되어 있다면, 실제 운영 전에는 Apps Script 스크립트 속성으로 이전하고 기존 키는 재발급하는 것을 권장합니다.

## 이후 개선 가능 항목

- 카카오 응답을 `simpleText`에서 `listCard`로 변경해 더 보기 좋은 목록형 말풍선으로 출력
- `배송완료 처리`, `고객 잔여회차 충전` 같은 쓰기 기능 추가
- 특정 관리자 카카오 userKey만 허용하는 화이트리스트 추가
- 로젠택배 복사용 포맷 별도 출력
